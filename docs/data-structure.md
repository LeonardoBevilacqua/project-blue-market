# Data structure

The ID fields should either be UUID or Integer.

- Kind of shopping
    - ID
    - Name: String
- Item type
    - ID
    - Kind ID
    - Name: String
- Item location
    - ID
    - Name: String
- Market list place
    - ID
    - Name: String
- Market list
    - ID
    - Name: String
    - Date: Date
    - Kind ID
    - Place ID
- Market list item
    - ID
    - Name: String
    - Quantity: Floating value
    - Unit value: Floating value
    - Total value: Floating value
    - Barcode: String
    - Created date: Date 
    - Modified date: Date 
    - Cloud modified date: Date (mobile app only)
    - Synchronized: Boolean (mobile app only)
    - Market list ID
    - Type ID
    - Location ID
