# Required data management

The market list and it items will require some data to be previously created, to later help us easily identify and filter the lists and items.
All of them should be mapped with at least an ID and the name.

## Kind of shopping

It should help to setup in which context the market list exists.

| ID | Name          |
|----|---------------|
| 1  | grocery       |
| 2  | pet store     |
| 3  | shopping mall |

## Item type

It should help to setup what the Market list item is.

| ID | Name    | Kind ID |
|----|---------|---------|
| 1  | rice    | 1       |
| 2  | juice   | 1       |
| 3  | pet toy | 2       |

## Item location

Optional data about where the item should be stored, usually used to sort the items.

| ID | Name         |
|----|--------------|
| 1  | kitchen      |
| 2  | bathroom     |
| 3  | service area |

## Market list place

Data about where the items were brought, to help with the filtering.

| ID | Name      |
|----|-----------|
| 1  | Tenda     |
| 2  | Savegnago |
| 3  | Max       |
